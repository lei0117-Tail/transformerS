# Add & Norm 完整流程演示（含 MHA + FFN 两个子层）

## 概念澄清

### ❌ 常见误解
> "FFN 处理之后的数据就是残差"

### ✅ 正确理解
- **FFN 的输出** = 对数据进行变换后的结果（不是残差）
- **残差 (Residual)** = 一种**连接方式**：`原始值 + 变换后的值`
- FFN 输出能和原始 x 相加的原因：**输入和输出维度相同**（都是 d_model 维）

---

## Encoder Layer 完整结构（两个子层）

```
输入 x
  │
  ├─► 子层一：Multi-Head Attention (MHA)
  │   │
  │   ▼
  │   LayerNorm → MHA(Q,K,V) → Add(残差) ──► 中间输出 x₁
  │                                          (= x + MHA输出)
  │
  ├─► 子层二：Feed Forward Network (FFN)
  │   │
  │   ▼
  │   LayerNorm → FFN(ReLU) → Add(残差) ──► 最终输出 x₂
  │                                       (= x₁ + FFN输出)
  │
  ▼
下一层 Encoder Layer
```

**关键点**：
- 每个 Encoder Layer 包含 **2 个子层**（Sublayer）
- 子层一 = **MHA**（注意力机制，token 之间交互信息）
- 子层二 = **FFN**（前馈网络，每个 token 独立做非线性变换）
- 每个子层都遵循：`LayerNorm → Sublayer → Residual Add`（Pre-LN 模式）

---

## 具体数值演示

### 场景设定

```
d_model = 4      （向量维度，方便展示）
d_ff    = 8      （FFN 内部放大倍数 = d_model × 2）
num_heads = 2    （注意力头数）
seq_len = 2      （句子有 2 个 token："猫"、"坐"）
```

---

### 第 0 步：输入 x（Word Embedding + Position Embedding）

```python
x = [[ 2.0, -1.0,  0.5,  3.0],    ← token "猫"
     [ 0.0,  1.5, -2.0,  0.8]]    ← token "坐"

形状: (batch=1, seq_len=2, d_model=4)
```

这些值已经包含了词嵌入和位置编码的信息。

---

### ═══ 子层一：Multi-Head Attention (MHA) ═══

#### 第 1 步：LayerNorm（归一化）

对每个 token 的向量独立归一化。以 token "猫" `[2.0, -1.0, 0.5, 3.0]` 为例：

```
均值 μ = (2.0 + (-1.0) + 0.5 + 3.0) / 4 = 1.125
标准差 σ ≈ 1.47

归一化后:
normed["猫"] = [(2.0-1.125)/1.47, (-1.0-1.125)/1.47,
                (0.5-1.125)/1.47,   (3.0-1.125)/1.47]
             = [ 0.59, -1.46, -0.42,  1.27]

normed["坐"] = [ 0.35,  1.12, -1.38, -0.09]

📌 数值范围从 [-2.0, 3.0] 收缩到 [-1.46, 1.27]，更稳定
```

#### 第 2 步：Multi-Head Attention（多头注意力）

MHA 让两个 token 之间**交换信息**：

```
normed 输入:
  "猫": [ 0.59, -1.46, -0.42,  1.27]
  "坐": [ 0.35,  1.12, -1.38, -0.09]

                ↓
         生成 Q, K, V 矩阵
                ↓
     计算注意力分数 (Q · Kᵀ / √d_k)
                ↓
         Softmax → 加权求和
                ↓

mha_out:
  "猫": [ 1.1, -0.8,  0.9,  0.3]   ← "猫"的表示融入了"坐"的信息!
  "坐": [-0.3,  0.7, -1.0,  0.5]   ← "坐"的表示融入了"猫"的信息!

📌 这是 MHA 的核心：token 之间互相"看"对方，交换信息
   处理前："猫"不知道"坐"是什么
   处理后："猫"的向量里包含了与"坐"的关系信息
```

#### 第 3 步：残差连接 Add（第一个残差）

```python
x1 = x + dropout(mha_out)   # 原始 + 注意力输出
```

```
原始 x:       [[ 2.0, -1.0,  0.5,  3.0],    ← "猫" (原始)
               [ 0.0,  1.5, -2.0,  0.8]]    ← "坐" (原始)

MHA 输出:     [[ 1.1, -0.8,  0.9,  0.3],
               [-0.3,  0.7, -1.0,  0.5]]

─────────────────────────────────────────────

中间输出 x₁:   [[ 3.1, -1.8,  1.4,  3.3],    ← "猫" = 原始 + 注意力增强
               [-0.3,  2.2, -3.0,  1.3]]    ← "坐" = 原始 + 注意力增强

📌 第一个残差完成！x₁ 将传入子层二（FFN）
```

---

### ═══ 子层二：Feed Forward Network (FFN) ═══

#### 第 4 步：LayerNorm（再次归一化）

对 x₁ 再次做 LayerNorm：

```
x₁["猫"] = [3.1, -1.8, 1.4, 3.3]

归一化后:
normed2["猫"] = [ 0.72, -1.21, -0.05,  1.53]
normed2["坐"] = [-0.22,  1.38, -1.25,  0.09]

📌 再次稳定数值，为 FFN 做准备
```

#### 第 5a 步：FFN 第一层线性变换（4维 → 8维，升维展开）

```
W₁ 形状: (4, 8)       b₁ 形状: (8,)

hidden = normed2["猫"] · W₁ + b₁
       = [1.2, -0.8, 0.3, 2.1, -1.5, 0.7, -0.2, 1.8]   ← 8 维！

📌 升维！从 4 维展开到 8 维
   类比：把一句话"展开"成更详细的描述
   给模型更多空间来学习复杂的特征组合
```

#### 第 5b 步：ReLU 激活函数（负数 → 0，特征选择）

```
ReLU 之前: [1.2, -0.8, 0.3, 2.1, -1.5, 0.7, -0.2, 1.8]
                    ↓     ↓      ↓     ↓   这 4 个是负数
ReLU 之后: [1.2,  0.0, 0.3, 2.1,  0.0, 0.7,  0.0, 1.8]

📌 特征选择机制！
   - 第 2/5/7 个维度被"关掉"（值为 0）
   - 只保留被"激活"的特征（正数部分）
   - 这就是非线性的来源：如果不用 ReLU，两层线性 = 一层线性
```

#### 第 5c 步：FFN 第二层线性变换（8维 → 4维，降维压缩）

```
W₂ 形状: (8, 4)       b₂ 形状: (4,)

ffn_out = ReLU(hidden) · W₂ + b₂
        = [0.8, -0.3, 1.1, 0.4]   ← 回到 4 维！

📌 降维！把高维空间的处理结果"压缩"回原始维度
   8 维中经过 ReLU 筛选后的特征，被融合浓缩到 4 维中
```

两个 token 都经过 FFN 后：

```
sublayer_out (FFN输出):
  "猫": [ 0.8, -0.3,  1.1,  0.4]
  "坐": [-0.2,  0.6, -0.5,  0.9]
```

#### 第 6 步：残差连接 Add（第二个残差）

```python
x2 = x1 + dropout(ffn_out)   # 子层一的输出 + FFN 输出
```

```
子层一输出 x₁: [[ 3.1, -1.8,  1.4,  3.3],    ← "猫" (经过 MHA 增强)
               [-0.3,  2.2, -3.0,  1.3]]    ← "坐" (经过 MHA 增强)

FFN 输出:      [[ 0.8, -0.3,  1.1,  0.4],
               [-0.2,  0.6, -0.5,  0.9]]

─────────────────────────────────────────────

最终输出 x₂:    [[ 3.9, -2.1,  2.5,  3.7],    ← "猫" = MHA增强 + FFN变换
               [-0.5,  2.8, -3.5,  2.2]]    ← "坐" = MHA增强 + FFN变换

📌 第二个残差完成！这就是一个完整 Encoder Layer 的最终输出
   x₂ 将传入下一个 Encoder Layer（如果有的话）
```

## 每一步的核心作用总结

| 步骤 | 子层 | 操作 | 核心作用 |
|------|------|------|---------|
| 1 | MHA | LayerNorm | 数值稳定化 |
| 2 | MHA | Multi-Head Attention | **Token 之间交换信息**（"猫"看到"坐"） |
| 3 | MHA | **Residual Add ①** | 原始信息 + 注意力增强信息 |
| 4 | FFN | LayerNorm | 再次数值稳定化 |
| 5a | FFN | 线性 4→8 | **升维展开**，获得更多表达空间 |
| 5b | FFN | ReLU | **非线性特征选择**，关闭不重要的维度 |
| 5c | FFN | 线性 8→4 | **降维压缩**，融合高维特征 |
| 6 | FFN | **Residual Add ②** | MHA 信息 + FFN 变换信息 |

---

## 为什么需要两个子层？

### 子层一 MHA 的独特价值
- **让 token 之间产生交互**
- "猫"可以关注"坐"，理解它们之间的关系
- 没有 MHA，每个 token 就是"孤岛"

### 子层二 FFN 的独特价值
- **每个 token 独立地做复杂非线性变换**
- 升维 → 非线性激活 → 降维
- 给模型能力去学习"每个位置应该怎么独立加工语义"

### 两者配合
```
MHA: "我和其他 token 是什么关系？"  → 关系层面
FFN: "我自己应该变成什么样？"        → 特征层面

两者缺一不可！就像：
- MHA = 开会讨论（大家交流意见）
- FFN = 各自消化（每个人回去独立思考）
```

## 关于"残差"的最简解释

```
残差 = x + Sublayer(x)

这不是说 Sublayer 的输出"变成了"残差，
而是说：我们用加法把「原始」和「变换后」融合在一起。

为什么叫"残差"？
因为梯度反向传播时：
  ∂Loss/∂x = ∂Loss/∂output × (1 + ∂Sublayer/∂x)
                                ^^
                          这个恒定的 1 确保
                          梯度不会消失！

就像一条"高速公路"，梯度可以直接穿过网络。
