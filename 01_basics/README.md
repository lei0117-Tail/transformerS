## 第一章：Embedding — 从 Token ID 到向量表示

### 🎯 这一章解决什么问题？

**计算机无法直接理解文字，必须把"词"变成"数"。**

Transformer 的输入是一个个整数（Token ID），但模型内部的所有计算都是矩阵运算。Embedding 层就是那个"翻译官"，负责把离散的整数 ID 映射成连续的向量表示。同时，因为 Transformer 没有 RNN 那样的时序结构（它并行处理所有 token），所以还需要显式地告诉模型 **每个 token 在句子中的位置**。

### 🔑 为什么要这么做？

| 需求 | 解决方案 | 为什么需要 |
|------|---------|-----------|
| 文字 → 数值 | **Token Embedding**（查表） | 神经网络只能处理数值，不能处理字符串 |
| 位置信息 | **Positional Encoding**（sin/cos 或可学习） | Transformer 并行处理所有 token，不知道谁在前谁在后 |
| 缩放平衡 | `× √d_model` | 防止位置编码的信号被 Embedding 的数值淹没 |

### 📁 对应文件

### `01_embedding.py` — 词嵌入 + 位置编码

| 类 | 作用 |
|----|------|
| `TokenEmbedding` | 把整数 token id 映射到 d_model 维连续向量（查找表） |
| `SinusoidalPositionalEncoding` | 用 sin/cos 函数注入绝对位置信息，**无需训练** |
| `LearnablePositionalEncoding` | 可学习的位置嵌入，BERT/GPT 风格 |
| `TransformerEmbedding` | 将 Token Embedding 与 Positional Encoding 相加，作为模型输入 |

**核心公式**：
```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

#### 📖 深入理解：从 Token ID 到 Embedding 输出的完整流程

以句子 `"我喜欢你"` 为例，假设 `seq_len=5`（需要补 2 个 pad），`d_model=64`：

##### 第一步：Token ID（分词 + 补 pad）

词表（vocab）是一个 `{词: id}` 的映射表，由分词器决定：

```python
vocab = {
    "<pad>": 0,     # 填充 token（占位符）
    "<unk>": 1,     # 未知词（没见过的词）
    "我":    10,
    "喜欢":  100,
    "你":    11,
}
```

输入变成一个整数序列：

```python
token_ids = [10, 100, 11, 0, 0]
#            我   喜欢  你  <pad> <pad>
#           shape: [batch_size=1, seq_len=5]
```

> **为什么需要 padding？**
> 一个 batch 里句子长短不一（比如有的3个词、有的5个词），必须补齐成相同长度才能拼成矩阵。
> 短句用 `<pad>` 填充到 `seq_len`。真正屏蔽 padding 的是后面 Attention 层中的 **mask**（把对应位置填成 `-∞`）。

##### 第二步：Token Embedding（查表）

`nn.Embedding(vocab_size, d_model)` 本质就是一个 **查找表** `[vocab_size × d_model]`：

```
         维度0   维度1   维度2   ...   维度63
id=0:   [-0.02, +0.15, -0.08, ..., +0.03]   ← <pad> 的向量
id=1:   [+0.05, -0.10, +0.22, ..., -0.01]   ← <unk> 的向量
...
id=10:  [+0.33, -0.45, +0.12, ..., +0.08]   ← "我"的向量
...
id=100: [-0.18, +0.27, -0.04, ..., +0.19]   ← "喜欢"的向量
...
id=11:  [+0.09, -0.31, +0.55, ..., -0.12]   ← "你"的向量
```

查完之后 shape 变为 `[1, 5, 64]`，然后乘以 `√d_model = √64 = 8` 放大（防止位置编码信号被淹没）。

##### 第三步：Positional Encoding（加位置信息）

位置编码是一个 **预计算的矩阵** `[max_len × d_model]`，用 sin/cos 公式算出。Token 向量与位置向量 **逐元素相加**：

```python
x = token_emb + pos_enc[:, :seq_len]
```

最终输出 shape：`[1, 5, 64]`，每个位置的向量都融合了**语义信息**和**位置信息**。

##### 完整数据流图

```
原始句子: "我喜欢你"
  ↓ 分词 + 补 pad
token ids: [10, 100, 11, 0, 0]          shape: [1, 5]
  ↓ nn.Embedding 查表（按 id 取对应行）
Token Emb:  [1, 5, 64]                  每个 token → 64维语义向量
  ↓ × √d_model（缩放）
  ↓ + Positional Encoding（逐元素相加）
最终输出:   [1, 5, 64]

  位置0: "我"的语义向量   + "位置0"编码 = 新向量A
  位置1: "喜欢"的语义向量 + "位置1"编码 = 新向量B
  位置2: "你"的语义向量   + "位置2"编码 = 新向量C
  位置3: <pad>的语义向量  + "位置3"编码 = 新向量D  ← 后续被 mask 屏蔽
  位置4: <pad>的语义向量  + "位置4"编码 = 新向量E  ← 后续被 mask 屏蔽
```

## 第二章：Attention — Transformer 的灵魂机制

### 🎯 这一章解决什么问题？

**如何让模型"关注"句子中重要的部分？**

在处理一句话时，不是所有词都同等重要。例如："这只**猫**坐在**地上**"，理解"猫"时需要关注"坐"和"地上"，而不需要关注不相关的词。Attention 机制就是让模型自动学习这种"关注度"分配。

### 🔑 为什么要这么做？

| 需求 | 解决方案 | 为什么需要 |
|------|---------|-----------|
| 区分重要性 | **Q·Kᵀ 注意力分数** | 让模型自己决定哪些词更重要 |
| 归一化权重 | **Softmax** | 把分数变成概率分布（和为 1），方便加权 |
| 防止数值过大 | **除以 √d_k** | 点积值过大时 softmax 会进入饱和区，梯度消失 |
| 屏蔽无效位置 | **Padding Mask / Causal Mask** | 忽略填充 token；防止 Decoder 偷看未来 |

### 📁 对应文件

### `02_attention.py` — 缩放点积注意力

Transformer 的核心计算单元。本文件实现最基础的单头注意力和两种 Mask。

| 函数/类 | 作用 |
|---------|------|
| `scaled_dot_product_attention` | 实现 `softmax(QKᵀ/√d_k)·V`，支持 mask 和 dropout |
| `make_padding_mask` | 生成填充掩码，屏蔽 `<pad>` 位置 |
| `make_causal_mask` | 生成下三角因果掩码，防止 Decoder 看到未来信息 |

**为什么要除以 √d_k**：防止向量维度大时点积值过大，导致 softmax 进入梯度饱和区。

## 第三章：Multi-Head Attention — 多角度理解序列

### 🎯 这一章解决什么问题？

**单头注意力只能学到一种"关注模式"，但语言的关系是多样的。**

"猫和狗在玩耍"这句话中：
- 从**语法**角度看："猫"应该关注"玩耍"（主谓关系）
- 从**语义**角度看："猫"应该关注"狗"（并列实体）
- 从**位置**角度看：相邻的词可能更相关

单头注意力只能学其中一种，多头注意力让模型**同时从多个子空间**学习不同类型的依赖关系。

### 🔑 为什么要这么做？

| 需求 | 解决方案 | 为什么需要 |
|------|---------|-----------|
| 多种关系类型 | **多个独立的 Attention 头** | 每个头可以专注不同模式（语法/语义/位置等） |
| 信息融合 | **W_O 线性投影** | 把多个头的输出合并回统一表示 |
| 不同使用场景 | **Q/K/V 的来源灵活配置** | Encoder 自注意力、Decoder 自注意力、交叉注意力 |

### 📁 对应文件

### `03_multi_head_attention.py` — 多头注意力

将单头注意力扩展为多头，让模型同时从不同子空间学习不同类型的依赖关系。

| 类 | 作用 |
|----|------|
| `MultiHeadAttention` | 核心多头注意力模块，包含 W_Q/W_K/W_V/W_O 四个线性变换 |

**三种使用场景**：
- **Encoder 自注意力**：`Q=K=V=x`，双向，无 Causal Mask
- **Decoder 自注意力**：`Q=K=V=x`，单向，带 Causal Mask
- **Decoder 交叉注意力**：`Q=decoder_out, K=V=encoder_out`

## 第四章：Feed Forward Network — 非线性特征变换

### 🎯 这一章解决什么问题？

**Attention 只做"信息收集"，FFN 才做"信息加工"。**

MHA 的本质是对 V 做加权平均（线性操作），它能把相关信息"找出来"，但不能创造新的特征组合。FFN 通过 **升维 → 非线性激活 → 降维** 的过程，让每个 token 独立地进行复杂的非线性变换——就像找到资料后需要"消化理解"一样。

### 🔑 为什么要这么做？

| 需求 | 解决方案 | 为什么需要 |
|------|---------|-----------|
| 引入非线性 | **ReLU / GELU / SwiGLU** | 没有非线性激活，两层 Linear 等价于一层 Linear，表达能力不够 |
| 特征选择 | **ReLU 把负数变 0** | 有选择地"关掉"不重要的特征维度，保留被激活的特征 |
| 增强表达能力 | **升维再降维 (4倍)** | 高维空间有更多自由度来学习复杂变换 |
| 保持维度一致 | **最终降回 d_model** | 输出必须和输入同维度，才能做后续的残差连接 |

### 📁 对应文件

### `04_feed_forward.py` — 位置前馈网络（FFN）

Encoder/Decoder 层的第二个子层。对序列中的**每个位置独立**做非线性变换。

| 类 | 激活函数 | 说明 |
|----|---------|------|
| `FeedForward` | ReLU 或 GELU | 标准 FFN，`d_model → d_ff → d_model` |
| `SwiGLUFeedForward` | SwiGLU | LLaMA 系列使用，门控机制，性能更好 |

#### 📖 FFN 公式逐项解析

**核心公式**：
```
FFN(x) = ReLU(x·W₁ + b₁)·W₂ + b₂
```

##### 所有参数都是可训练的

`nn.Linear` 内部自动创建 4 个可训练参数，全部通过反向传播从数据中学习：

| 参数 | 对应代码 | 形状（以 d_model=128, d_ff=512 为例） | 初始化方式 |
|------|---------|---------------------------------------|-----------|
| **W₁** | `linear1.weight` | `[512, 128]` | 随机（Xavier/Kaiming） |
| **b₁** | `linear1.bias` | `[512]` | 全 **0** |
| **W₂** | `linear2.weight` | `[128, 512]` | 随机 |
| **b₂** | `linear2.bias` | `[128]` | 全 **0** |

##### 具体计算流程（以 d_model=4, d_ff=8 为例）

**输入 x**: 来自 Multi-Head Attention 的输出

```
输入 x:     [2,   5, 128]                    ← MHA 的输出
  │
  ▼
┌─────────────────────┐
│  Linear(d_ff=512)   │  W₁: [512, 128], b₁: [512]
│   x · W₁ + b₁       │
└──────────┬──────────┘
           │
           ▼  shape: [2, 5, 512]  ⬆️ 升维 4 倍！
           │
┌─────────────────────┐
│       ReLU          │  负数→0，正数不变（引入非线性！）
└──────────┬──────────┘
           │
           ▼  shape: [2, 5, 512]
           │
┌─────────────────────┐
│  Linear(d_model=128)│  W₂: [128, 512], b₂: [128]
│   (···)·W₂ + b₂     │
└──────────┬──────────┘
           │
           ▼  shape: [2, 5, 128]  ⬇️ 降回原维度 ✅

输出:      [2,   5, 128]                   ← 送入下一个 Add & Norm
```

##### FFN 存在的核心目的

**一句话总结**：MHA 负责"从上下文中提取信息"，FFN 负责"加工和消化这些信息"。

| 组件 | 类比 | 做了什么 |
|------|------|---------|
| **Multi-Head Attention** | 👀 翻书查阅 | "这句话和前面哪些内容相关？把相关信息找出来" |
| **Feed Forward Network** | 🧠 思考消化 | "找到的信息，我该怎么理解、怎么转换、怎么输出？" |

###### 为什么光有 Attention 不够？

1. **Attention 本质是线性的**：输出 = Σ(权重 × V)，只是对 V 做加权平均，无法创造新的特征组合
2. **FFN 引入非线性**：中间的 ReLU 让网络学会选择性激活和特征重组
3. **升维再降维**：128 → 512 → 128，高维空间有更多自由度来学习复杂变换

##### Position-wise（逐位置）的含义

FFN 最重要的一点：**每个 token 独立处理，token 之间不交互！**

```
Token 0: [128] → Linear → ReLU → Linear → [128]   ✅ 独立
Token 1: [128] → Linear → ReLU → Linear → [128]   ✅ 独立（共享参数）
Token 2: [128] → Linear → ReLU → Linear → [128]   ✅ 独立
...
```

相当于对每个位置用同一个 MLP（多层感知机），但位置之间没有任何信息交流。

##### 参数量对比（以 d_model=128, d_ff=512 为例）

一个 Encoder Layer 的参数分布：

```
┌─────────────────────────┬──────────┐
│  Multi-Head Attention   │  ~33%    │  ← Q/K/V/O 投影
│  Feed Forward Network   │  ~67%    │  ← W₁/b₁/W₂/b₂  👈 大头在这
│  LayerNorm              │  <1%     │  ← γ, β
└─────────────────────────┴──────────┘

具体数值：
  W₁: 512 × 128 = 65,536
  b₁:         512 =     512
  W₂: 128 × 512 = 65,536
  b₂:         128 =     128
  ──────────────────────
  总计:            131,712  （约 MHA 的 2 倍）
```

> 这就是为什么大模型动辄几十亿、几百亿参数 —— **大部分都在 FFN 层里**！

##### 🔑 关键疑问：LayerNorm 输出均值≈0（有正有负），到 FFN 被 ReLU 把负数变 0，信息不就丢失了吗？

**这正是 ReLU 的设计目的 —— 有选择地"关掉"一部分特征。**

###### ReLU 不是在"破坏信息"，而是在做"特征选择"

假设 LayerNorm 输出一个 token 的向量（简化为 6 维）：

```python
# LayerNorm 输出（均值≈0，标准差≈1）
x = [1.5, -0.3, 2.1, -1.8, 0.4, -0.9]
      ↑     ↑     ↑     ↑     ↑     ↑
     "名词" "否定" "动词" "时态" "数"  "语气"

# 经过 ReLU 后
ReLU(x) = [1.5,  0,   2.1,  0,    0.4,  0 ]
              ✅保留    ❌关掉  ✅保留  ❌关掉  ✅保留  ❌关掉
```

类比大脑读句子 **"猫坐在地上"**：
- 处理 **"猫"** 时：需要激活"动物、名词、主语"特征 ✅
- 不需要激活："动词、否定、过去时" 特征 ❌ → ReLU 自动关掉

###### 但纯 ReLU 有缺陷 —— 现代模型的改进

如果某个重要特征的值恰好是负数，ReLU 会把它**永久归零**。这就是为什么现代模型不再用纯 ReLU：

| 激活函数 | 负数处理 | 谁在用 |
|---------|---------|--------|
| **ReLU** | 直接变 **0**（硬切断） | 原始 Transformer |
| **GELU** | 平滑过渡到 **~0**（不完全归零） | GPT-2, GPT-3, BERT |
| **SwiGLU** | 门控机制，**选择性通过** | LLaMA, PaLM |

```
输入:    [-2.0,  -1.0,  -0.5,  0.0,  0.5,  1.0,  2.0]

ReLU:    [ 0.0,   0.0,   0.0,  0.0, 0.5, 1.0, 2.0]   ← 负数全丢
GELU:    [-0.045, -0.159, -0.155, 0.0, 0.341, 0.841, 1.995]  ← 平滑过渡
```

###### 更关键：FFN 最终输出又有正有负了！

虽然中间 ReLU 会把负数变 0，但 **FFN 最后一步 `·W₂ + b₂` 是线性变换**，输出不受限制：

```python
# ReLU 输出（非负）
relu_out = [1.5, 0, 2.1, 0, 0.4, 0, ..., 0, 3.2]  # 全 ≥ 0

# 经过 W₂ + b₂（线性变换，可正可负）
final_out = relu_out @ W₂ + b₂
# = [0.9, 0.7, -0.2, 0.4, ...]  ← 又有正有负了！
```

所以下一层的 LayerNorm 又能正常工作（均值≈0），形成闭环：

```
LayerNorm → 有正有负(均值≈0)
     ↓
FFN: Linear→ReLU(负变0)→Linear → 又恢复有正有负
     ↓
下一层 LayerNorm → 正常工作（均值≈0）
     ↓
... 循环 ...
```

## 第五章：Add & Norm — 训练稳定的基石

### 🎯 这一章解决什么问题？

**深层网络有两个致命问题：梯度消失和信息丢失。Add & Norm 就是来解决这两个问题的。**

想象一下信息经过 6 层、12 层甚至 96 层的网络：
- **没有 Norm**：每层数值范围可能越来越大或越来越小，最终爆炸或消失
- **没有 Add**：梯度反向传播时要乘以每一层的导数，连乘多次后趋近于 0（梯度消失）

### 🔑 为什么要这么做？

| 需求 | 解决方案 | 为什么需要 |
|------|---------|-----------|
| 数值稳定 | **LayerNorm 归一化** | 把每层输出的均值拉回 0、方差拉回 1，防止数值爆炸/消失 |
| 梯度流通 | **Residual Connection（残差）** | `x + F(x)` 的导数恒有 +1 项，梯度直达浅层 |
| 信息融合 | **原始 + 变换后** | 最终输出 = 原始信息 + 新信息，不会丢掉基础表示 |
| 训练稳定 | **Pre-LN（先 Norm 后子层）** | 现代主流方式，梯度路径更短更稳，无需 warm-up |

### 📁 对应文件

### `05_layer_norm.py` — 层归一化 + 残差连接（Add & Norm）

让深层 Transformer 训练稳定的两个关键技术。Transformer 中每个子层后面都跟着 Add & Norm。

| 类 | 作用 |
|----|------|
| `LayerNorm` | 对每个 token 的特征维度做归一化（区别于 BatchNorm） |
| `ManualLayerNorm` | 手动实现 LayerNorm，帮助理解原理 |
| `PostLNSublayer` | Post-LN：`LayerNorm(x + Sublayer(x))`，原版 Transformer（2017） |
| `PreLNSublayer` | Pre-LN：`x + Sublayer(LayerNorm(x))`，GPT-2 以后的标准（现代主流） |

#### 📖 核心概念：什么是"子层"（Sublayer）？

一个 Encoder Layer 内部有 **两个子层**，每个子层都被一组 Add & Norm 包裹：

```
┌─────────────────────────────────────────┐
│           Encoder Layer（一层）          │
│                                         │
│   ┌───────────────────────────────┐     │
│   │  子层1 (Sublayer 1)            │     │
│   │  = Multi-Head Attention       │     │
│   └───────────────────────────────┘     │
│             ↓ Add & Norm                │
│   ┌───────────────────────────────┐     │
│   │  子层2 (Sublayer 2)            │     │
│   │  = Feed Forward Network       │     │
│   └───────────────────────────────┘     │
│             ↓ Add & Norm                │
└─────────────────────────────────────────┘
```

> "子层"就是 Encoder Layer 内部真正干活的两个计算单元。外层的 Add & Norm 包装对两个子层是完全一样的，只是里面的计算不同。

#### 🔀 架构演变：Post-LN vs Pre-LN

**核心区别就一个：Norm 在 Add 之前还是之后？**

```
═══════════════════════════════════════════════════
  Post-LN（原论文 2017）— 层后归一化
═══════════════════════════════════════════════════

  x → [Sublayer] → [Dropout] → [+ x] → [LayerNorm] → 输出
       子层处理              ↑ Add       ↑ 最后才Norm

  公式: output = LayerNorm(x + Sublayer(x))
  ⚠️ 问题: 梯度必须穿过 LayerNorm，深层时容易不稳定，需要 warm-up


═══════════════════════════════════════════════════
  Pre-LN（现代主流 2018+）— 层前归一化（本项目默认）
═══════════════════════════════════════════════════

  x → [LayerNorm] → [Sublayer] → [Dropout] → [+ x] → 输出
       ↑ 先Norm       子层                ↑ 后Add（不再Norm）

  公式: output = x + Sublayer(LayerNorm(x))
  ✅ 优势: 梯度直接通过 Add 流过（恒有 +1 项），训练更稳定，无需 warm-up
  代表模型: GPT-2, GPT-3, BERT, LLaMA, 所有现代大模型
```

| 对比项 | Post-LN（原论文） | Pre-LN（现代主流） |
|--------|-------------------|-------------------|
| **公式** | `Norm(x + F(x))` | `x + F(Norm(x))` |
| **Norm 位置** | Add **之后** | Add **之前** |
| **输出是否归一化** | ✅ 最终输出归一化了 | ❌ 最终输出**没有**归一化 |
| **需要 warm-up** | ✅ 是 | ❌ 不需要 |
| **梯度路径** | 梯度必须穿过 LayerNorm（可能被压缩） | 梯度直接通过 Add（恒有 +1） |
| **训练稳定性** | ⚠️ 深层时容易不稳定 | ✅ 非常稳定 |
| **适用深度** | 6 层左右还行 | 80 层以上也能稳定训练 |
| **代表模型** | 原始 Transformer | GPT-2/3, BERT, LLaMA 等 |

#### 🗺️ Encoder Layer 完整路线图（Pre-LN 版本）

涉及到的 **三种关键向量**：

| 向量 | 来源 | 含义 |
|------|------|------|
| **① 原始向量** | Embedding 层 / 上一层输出 | Token Emb + Position Emb |
| **② 注意力向量** | Multi-Head Attention 输出 | 收集了上下文信息后的向量 |
| **③ FFN 向量** | Feed Forward Network 输出 | 经过非线性加工后的向量 |

**完整数据流（以 shape `[2, 12, 128]` 为例）：**

```
输入: ① 原始向量 x  [2, 12, 128]
      来自 Embedding 或上一层 Encoder Layer
      │
      ├── ╔══ 第一套 Add & Norm（包裹 子层1: MHA）══ ╗──┐
      │    ║                                              ║  │
      │    ║  residual = x         # 保存 ① 原始向量      ║  │
      │    ║  x = Norm1(x)         # 先归一化             ║  │
      │    ║  x = MHA(Q=x,K=x,V=x) # 多头自注意力          ║  │
      │    ║  x = Dropout(x)                            ║  │
      │    ║  x = residual + x     # Add: ① + ②注意力     ║  │
      │    ║                                              ║  │
      │    ║  ↓ 中间结果（融合了原始信息 + 注意力信息）     ║  │
      │    ╚══ 第二套 Add & Norm（包裹 子层2: FFN）══ ═══╝  │
      │                                                   │
      │    residual = x          # 保存中间结果            │
      │    x = Norm2(x)          # 先归一化               │
      │    x = FFN(x)            # 前馈网络               │
      │    x = Dropout(x)                               │
      │    x = residual + x      # Add: 中间 + ③FFN       │
      │                                                   │
      ├───────────────────────────────────────────────────┘
      ▼
输出: [2, 12, 128]  ← shape 不变！可无限堆叠
（包含: 原始信息 + 注意力信息 + FFN 加工信息）
      │
      └────→ 送入下一层 Encoder Layer（或 Decoder）
```

#### 📊 具体数值演示（完整走一遍两个子层）

> 场景设定：`d_model=4`, `d_ff=8`, `seq_len=2`（token "猫"、"坐"）

##### 第 0 步：输入 x（Word Embedding + Position Embedding）

```
x = [[ 2.0, -1.0,  0.5,  3.0],    ← token "猫"
     [ 0.0,  1.5, -2.0,  0.8]]    ← token "坐"

形状: (batch=1, seq_len=2, d_model=4)
```

##### ═══ 子层一：Multi-Head Attention (MHA) ═══

**Step 1 — LayerNorm（归一化）：为什么需要？**
> 防止数值爆炸/消失。经过 MHA 和 FFN 后，数值范围可能越来越大，归一化把均值拉回 0、方差拉回 1，让每层的输入都在稳定的范围内。

```
输入 "猫": [ 2.0, -1.0,  0.5,  3.0]    范围: [-2.0, 3.0]
归一化后:   [ 0.59, -1.46, -0.42,  1.27]  范围: [-1.46, 1.27]  ✅ 更集中稳定
输入 "坐": [ 0.0,  1.5, -2.0,  0.8]
归一化后:   [ 0.35,  1.12, -1.38, -0.09]
```

**Step 2 — Multi-Head Attention：为什么需要？**
> 让 token 之间交换信息。"猫"可以关注"坐"，理解它们之间的关系。没有 MHA，每个 token 就是"孤岛"。

```
mha_out:
  "猫": [ 1.1, -0.8,  0.9,  0.3]   ← "猫"融入了"坐"的信息!
  "坐": [-0.3,  0.7, -1.0,  0.5]   ← "坐"融入了"猫"的信息!
```

**Step 3 — 残差连接 Add ①：为什么需要？**
> 三大作用：① 梯度高速公路（导数恒有+1）；② 子层没学到时保留原始信息；③ 融合新旧信息。

```
原始 x:       [[ 2.0, -1.0,  0.5,  3.0],
               [ 0.0,  1.5, -2.0,  0.8]]
MHA 输出:     [[ 1.1, -0.8,  0.9,  0.3],
               [-0.3,  0.7, -1.0,  0.5]]
─────────────────────────────────────────────
中间输出 x₁:   [[ 3.1, -1.8,  1.4,  3.3],    ← 原始 + 注意力增强
               [-0.3,  2.2, -3.0,  1.3]]
```

##### ═══ 子层二：Feed Forward Network (FFN) ═══

**Step 4 — LayerNorm（再次归一化）**

```
x₁["猫"] = [3.1, -1.8, 1.4, 3.3]
归一化后:   [ 0.72, -1.21, -0.05,  1.53]
```

**Step 5a — FFN 线性变换 ①（4维 → 8维，升维展开）：为什么需要？**
> 高维空间有更多"自由度"来学习复杂的特征组合。就像从"一行笔记"扩展成"一篇详细报告"。

```
hidden = [1.2, -0.8, 0.3, 2.1, -1.5, 0.7, -0.2, 1.8]   ← 8 维！
```

**Step 5b — ReLU（负数 → 0）：为什么需要？**
> 引入非线性 + 特征选择。只保留被"激活"的特征，关掉不重要的维度。没有非线性，两层 Linear = 一层 Linear。

```
ReLU 前: [1.2, -0.8, 0.3, 2.1, -1.5, 0.7, -0.2, 1.8]
ReLU 后: [1.2,  0.0, 0.3, 2.1,  0.0, 0.7,  0.0, 1.8]
                        ↑     ↑      ↑     ↑   这 4 个被关掉了
```

**Step 5c — FFN 线性变换 ②（8维 → 4维，降维压缩）：为什么需要？**
> 必须回到原始 d_model 维度，才能和原始向量做残差相加。同时把高维空间的处理结果"浓缩"融合。

```
ffn_out = [0.8, -0.3, 1.1, 0.4]   ← 回到 4 维！✅
```

**Step 6 — 残差连接 Add ②**

```
子层一输出 x₁: [[ 3.1, -1.8,  1.4,  3.3],
               [-0.3,  2.2, -3.0,  1.3]]
FFN 输出:      [[ 0.8, -0.3,  1.1,  0.4],
               [-0.2,  0.6, -0.5,  0.9]]
─────────────────────────────────────────────
最终输出 x₂:    [[ 3.9, -2.1,  2.5,  3.7],    ← MHA增强 + FFN变换
               [-0.5,  2.8, -3.5,  2.2]]
```

#### ⚠️ 关键概念澄清：FFN 输出 ≠ 残差！

> 这是一个非常常见的误解，必须纠正：

| 说法 | 对/错 | 解释 |
|------|-------|------|
| "FFN 处理之后的数据就是残差" | ❌ | FFN 输出只是变换后的结果 |
| "`x + FFN输出` 才是残差连接的结果" | ✅ | **加法操作**才是残差 |
| "残差是一种连接方式，不是某个模块的属性" | ✅ | MHA 和 FFN 都可以用残差连接 |

**为什么 FFN 的输出能和原始 x 相加？**

```
输入 x:  (batch, seq, d_model=4)  →  4 维向量
FFN输出: (batch, seq, d_model=4)  →  4 维向量   ← 关键！维度相同！

虽然 FFN 内部经历了 4 → 8 → 4 的变换，
但最终"必须回到原维度"，所以可以逐元素相加。
```

**生活类比**：你给厨师一个苹果（输入），厨师切片→加调料→摆盘，最后端出来的还是一盘苹果菜（输出）。虽然中间处理了很多步，但"苹果"这个主体没变，所以可以和原来的苹果"叠加"（融合原味和新味）。

#### ⚠️ 原始向量不需要单独传递！

Add 的输出已经融合了原始信息：

```
第1层: ①原始[3.0,1.5] → MHA→②[0.5,-0.3] → Add→[3.5,1.2] → FFN→③[0.2,0.8] → Add→[3.7,2.0]
                                                        ↑
                                              这个输出包含了一切！

第2层: 输入=[3.7,2.0]  （直接用上一层的输出，不需要额外传 ①）
       ↑
       这里面已经编码了最初的 [3.0,1.5] 信息 + 两轮加工的新信息

就像记账:
  第1层结束: 本金100 + 利息5 = 105   ← 105 里有 100
  第2层结束: 本金105 + 利息8 = 113   ← 113 里有 105（也就有最初的 100）
  第3层结束: 本金113 + 利息3 = 116   ← 116 里有全部历史
```

#### 📖 深入理解：Add & Norm 的两个部分

##### Norm（层归一化 LayerNorm）

**公式**：`y = (x - mean) / √(var + ε) × γ + β`

对**每个 token 的特征维度**独立归一化（不跨样本，区别于 BatchNorm）：

```python
mean = x.mean(dim=-1, keepdim=True)           # 均值 → [B, S, 1]
var  = ((x - mean) ** 2).mean(dim=-1, keepdim=True)  # 方差
x_norm = (x - mean) / torch.sqrt(var + eps)   # 归一化 → 均值≈0, 标准差≈1
output = gamma * x_norm + beta                 # 可学习缩放和偏移
```

- `γ` 初始化为 **1**，`β` 初始化为 **0**（可学习参数）
- **为什么需要？** 防止数值爆炸/消失，稳定训练；不依赖 batch size，适合变长序列

##### Add（残差连接 Residual Connection）

**公式**：`output = 原始输入 + 子层输出`（代码就是一个 `+` 号）

```python
# Pre-LN 方式的完整流程（5 行代码）
residual = x              # 第1行: 保存原始向量（"拍照留底"）
x = self.norm1(x)         # 第2行: 归一化（准备阶段，让子层输入稳定）
x = self.self_attn(...)   # 第3行: 子层处理 ← 核心计算，产生新信息
x = self.dropout(x)       # 第4行: 随机置零（防过拟合）
x = residual + x          # 第5行: 🎯 残差！原始 + 子层输出
```

⚠️ **重要理解**：加的是**子层的输出**，不是归一化的值！Norm 和 Dropout 都是辅助操作，真正产生新信息的是子层（MHA 或 FFN）。残差把子层的产出叠加到原始上。

**三大作用**：
1. **梯度高速公路**：`+` 的导数恒为 **1**，梯度直达浅层，防止消失（`∂(x+F)/∂x = 1 + ∂F/∂x`）
2. **保证最低性能**：子层没学到时（输出≈0），`x + 0 = x`，原始信息原样保留
3. **信息融合**：最终输出 = 原始信息 + 新信息，不需要单独传递原始向量

##### 为什么需要两个子层？

```
MHA: "我和其他 token 是什么关系？"  → 关系层面（token 之间交互）
FFN: "我自己应该变成什么样？"        → 特征层面（每个 token 独立加工）

两者缺一不可！就像：
- MHA = 开会讨论（大家交流意见）
- FFN = 各自消化（每个人回去独立思考）
```

##### 完整总结：一句话记忆

> **Norm 让数值稳定，Add 让梯度流通；Norm 是准备工作，加的是子层的产出；Pre-LN 先 Norm 后 Add，现代大模型都在用。**

##### 常见问题

| 问题 | 答案 |
|------|------|
| LayerNorm vs BatchNorm？ | LN 对每个 token 特征维归一化；BN 跨样本归一化。Transformer 用 LN 因为序列长度不固定 |
| γ 和 β 是什么？ | 可学习参数，初始化 γ=1, β=0。相当于"先不做任何事"，让网络自己决定是否需要调整 |
| 残差为什么能防梯度消失？ | `∂(x+F(x))/∂x = 1 + ∂F/∂x`，恒有 **1** 这一项，梯度至少不衰减 |
| 一个 Encoder Layer 有几个 LayerNorm？ | **2 个**：norm1（MHA 前）、norm2（FFN 前） |
| 加的是归一化的值还是子层输出？ | **子层输出！** Norm 只是准备阶段，真正要加的是 MHA 或 FFN 处理后的结果 |
| 原始向量需要单独传给下一层吗？ | **不需要！** Add 已经把原始信息编码进输出了（像记账一样累积） |
| FFN 输出就是残差吗？ | **不是！** FFN 输出只是变换结果，残差是 `x + FFN输出` 这个**加法操作**创造的 |

## 第六章：Encoder Layer — 完整组装

### 🎯 这一章解决什么问题？

**把前面 5 个模块像搭积木一样组装成一个完整的 Transformer 编码器层。**

前面我们分别学习了 Embedding、Attention、MHA、FFN、Add&Norm 这 5 个基础组件。这一章要把它们按照 Transformer 的架构"拼装"起来，形成一个功能完整的 Encoder Layer。这个 Encoder Layer 可以**无限堆叠**（6层、12层、96层...），构成完整的 Transformer Encoder。

### 🔑 为什么要单独封装成一个类？

| 需求 | 解决方案 | 为什么需要 |
|------|---------|-----------|
| 模块复用 | **EncoderLayer 类** | 定义一次，堆叠 N 次即可得到 N 层编码器 |
| 接口统一 | **标准 forward(x, mask)** | 不管内部多复杂，对外只暴露简单的输入输出接口 |
| Shape 保证 | **输入输出都是 [B, S, D]** | 只要维度不变，就可以任意深度堆叠 |

### 📁 对应文件

### `06_encoder_layer.py` — 完整 Encoder 层

将前5步的所有模块组合成一个完整的 Transformer Encoder Layer。

| 类 | 作用 |
|----|------|
| `EncoderLayer` | 完整 Encoder 层：MHA → Add&Norm → FFN → Add&Norm |

**数据流**：
```
输入 x
  → [多头自注意力] → 残差+归一化
  → [前馈网络]     → 残差+归一化
→ 输出（形状不变，仍为 [batch, seq_len, d_model]）
```

**代码层面的对应关系**：

```python
class EncoderLayer(nn.Module):
    def __init__(self, d_model, num_heads, d_ff, dropout):
        self.self_attn = MultiHeadAttention(d_model, num_heads)   # 第三章
        self.ffn = FeedForward(d_model, d_ff, dropout)            # 第四章
        self.norm1 = LayerNorm(d_model)                           # 第五章
        self.norm2 = LayerNorm(d_model)                           # 第五章
        self.dropout = nn.Dropout(dropout)                         # 第五章

    def forward(self, x, mask):
        # ===== 子层一：MHA + Add & Norm =====
        residual = x                              # 保存原始
        x = self.norm1(x)                         # Pre-Norm
        x = self.self_attn(x, x, x, mask)         # 注意力（子层）
        x = self.dropout(x)
        x = residual + x                          # 残差 Add

        # ===== 子层二：FFN + Add & Norm =====
        residual = x                              # 保存中间结果
        x = self.norm2(x)                         # Pre-Norm
        x = self.ffn(x)                           # 前馈网络（子层）
        x = self.dropout(x)
        x = residual + x                          # 残差 Add

        return x   # shape 不变: [batch, seq_len, d_model]
```

### 📊 六章知识总览

| 章节 | 模块 | 解决的问题 | 核心贡献 | 类比 |
|------|------|-----------|---------|------|
| **一** | Embedding | 文字→数值 | 把 token 变成向量 + 注入位置 | **身份证**（给每个词发一张带位置的证件） |
| **二** | Attention | 谁重要？ | 计算 token 之间的关注度分配 | **聚光灯**（照亮重要的词） |
| **三** | MHA | 多角度关注 | 同时学习多种关系类型 | **多盏灯**（从不同角度打光） |
| **四** | FFN | 加工信息 | 非线性特征变换（升维→激活→降维） | **消化思考**（找到资料后深入理解） |
| **五** | Add & Norm | 稳定训练 | 数值稳定 + 梯度流通 + 信息融合 | **安全网 + 高速公路** |
| **六** | Encoder Layer | 完整组装 | 把所有模块拼成可堆叠的标准层 | **完整工厂流水线** |
